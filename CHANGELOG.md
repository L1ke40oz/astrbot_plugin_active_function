# Changelog

## v2.3.0 (2026-06-29)

### 新增
- **可配置的引用回复标签格式**：新增 `reply_tag` 配置项，支持自定义引用标签格式（如 `<reply:{id}>`、`【reply:{id}】`等）
- **兜底标签清理机制**：添加低优先级（priority=5）的兜底清理器，防止所有控制标签泄漏给用户
- **智能标签识别**：支持不区分大小写和容错负数 ID 的标签匹配，自动清理常见变体

### 改进
- **提示词增强**：在引用回复提示词中明确要求使用小写 `reply` 和正整数 ID
- **更强的错误处理**：自动识别并清理格式错误的标签（如 `[Reply:123]`、`[reply:-123]` 等）
- **日志优化**：初始化时输出 `reply_tag` 配置，便于调试
- **多层防护**：主识别层（精确匹配）+ 兜底清理层（容错匹配），确保标签不泄漏

### 修复
- 修复插件禁用时标签仍可能泄漏的问题
- 修复 LLM 输出大写或负数 ID 时标签未清理的问题
- 修复非支持平台上标签泄漏的问题

### 技术细节
- `ReplyManager` 支持动态正则生成（`_build_reply_pattern`）
- 新增 `strip_all_reply_variants` 方法清理所有已知变体
- 所有硬编码的 `\[reply:(\d+)\]` 正则已替换为动态版本
