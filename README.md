# astrbot_plugin_active_function

为 AstrBot 提供主动能力的插件：消息撤回、引用回复、戳一戳。

当前仅支持 aiocqhttp（OneBot v11）平台私聊。

## 功能概览

### 消息撤回

Bot 可以在回复中使用 `[recall]` 标签，标记的那一段消息会在发送后自动撤回（默认 5 秒）。

```
正常的回复内容
[recall]其实我觉得你很可爱
后续正常内容
```

效果：第二段会短暂展示后自动撤回，其余内容正常保留。

### 引用回复

Bot 可以使用 `[reply:ID]` 标签引用用户之前发送的消息，系统会将其转换为 QQ 原生引用回复。

```
[reply:12345]这条消息说得对
```

- 系统会在消息防抖合并前缓存每条原始消息的 ID
- LLM 的 system prompt 中会注入可引用的消息列表
- 只有列表中存在的 ID 才能被引用

### 戳一戳

**接收戳一戳**：用户戳 Bot 时，会将戳一戳事件转换为文本注入到正常的 LLM 对话流程中（经过消息防抖合并），Bot 可以自然地回应。

**发送戳一戳**：Bot 可以在回复中使用 `[poke]` 标签，系统会在消息发送前执行戳一戳动作。

```
在做什么 [poke]
```

## 标签与对话历史

**不变量**：控制标签（`[recall]`、`[reply:ID]`、`[poke]`）**永远只在发送给用户的消息中被过滤**——标签在 `on_decorating_result` 阶段从最终消息链里剥离并触发对应动作，用户始终看不到它们。

至于这些标签**如何写入对话历史/对话数据**，可通过 `history.tag_mode` 配置在三种模式中选择（不影响发送给用户的内容）：

| 模式 | 行为 | 适用场景 |
| --- | --- | --- |
| `strip` | 从历史中**剥离**全部标签，对话记录干净 | 不希望对话数据里出现任何标签 |
| `keep` | **原样保留**标签 | 想让对话数据里保留原始标签文本 |
| `annotate`（默认） | 把标签替换成**可自定义的摘要文字** | 既想让 AI 记得自己引用/撤回了什么，又不想让历史堆满原始标签 |

### 为什么默认用 `annotate`

- `keep` 虽然能让模型看到自己用过的标签，但说几句话就会堆满 `[reply:...]` / `[poke]` 噪音，污染上下文；且历史里的 `[reply:12345]` 只是个数字，引用缓存（默认 600 秒 TTL）过期后该 ID 已不在可引用列表中，模型可能**照抄失效的旧 ID**，与"只能用列表里的 ID"的提示词冲突。
- `annotate` 把标签转写成模型读得懂的自然语言，例如：
  - `[reply:12345]我同意` → `（引用了「今天天气真好啊」）我同意`
  - `[recall]其实你很笨` → `（接下来这句已被撤回）其实你很笨`
  - `[poke]` → `（戳了戳对方）`

### 自定义摘要模板

`annotate` 模式下每种标签的摘要文字都可在配置中自定义（这些编辑项只在选择 `annotate` 时显示）：

- `history.annotate_reply_template`：默认 `（引用了「{quote}」）`。占位符 `{quote}` = 被引用消息内容（截断 30 字，缓存过期则为空），`{id}` = 被引用消息 ID。
- `history.annotate_recall_template`：默认 `（接下来这句已被撤回）`（被撤回的正文仍保留在历史里）。
- `history.annotate_poke_template`：默认 `（戳了戳对方）`。

> 设计参考了 `astrbot_plugin_proactive_message` 对 `[NEXT]` 标签的处理方式：标签从输出里过滤掉，但在对话记录中保留信息。

## 配置说明

安装后在 AstrBot 管理面板中配置。

| 配置项 | 说明 |
|--------|------|
| **分段正则表达式** | 用于分割 Bot 回复为多条消息的正则（全局生效）。留空不启用分段。 |
| **分段发送间隔** | 分段发送时每条消息之间的间隔时间（秒） |

### 消息撤回配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 启用撤回功能 | `true` | 总开关 |
| 撤回延迟 | `5` 秒 | 消息发送后多久执行撤回 |
| 撤回标签 | `[recall]` | LLM 回复中的撤回标记 |

### 引用回复配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 启用引用回复 | `true` | 总开关 |
| 消息缓存有效期 | `600` 秒 | 超时后消息不再可被引用 |
| 每会话最大缓存数 | `50` | 每个用户会话最多缓存多少条消息 |

### 戳一戳配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 启用戳一戳 | `true` | 总开关 |
| 冷却时间 | `5` 秒 | 同一用户连续戳的最小间隔 |
| 提示词模板 | 见下方 | 用户戳 Bot 时注入的提示词，`$username` 会被替换 |
| 戳一戳标签 | `[poke]` | LLM 回复中的戳一戳标记 |

### 提示词注入配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 启用提示词注入 | `true` | 总开关 |
| 撤回功能提示词 | 内置模板 | 告知 LLM 如何使用 `[recall]` |
| 引用回复提示词 | 内置模板 | 告知 LLM 如何使用 `[reply:ID]` |
| 戳一戳功能提示词 | 内置模板 | 告知 LLM 如何使用 `[poke]` |

## 工作原理

### 事件处理优先级

```
priority 100: 缓存用户消息（在防抖合并前）
priority  90: 拦截戳一戳事件，注入文本
priority  13: [TTS 插件] 处理 <tts> 标签，转换为 Record 语音组件
priority  12: 处理 [poke] 标签（剥离 + 执行动作）
priority  11: 处理 [reply:ID] 标签（接管发送，剥离 <tts> 残留标签）
priority  10: 处理 [recall] 标签（接管发送 + 定时撤回，剥离 <tts> 残留标签）
```

### 标签生命周期

```
LLM 输出 → on_llm_response（保存原始文本到 event extra；按 history.tag_mode
            将写入历史的文本做 strip/keep/annotate 处理，发送内容不受影响）
         → 对话历史保存（取决于 tag_mode：剥离 / 原样保留 / 摘要）
         → on_decorating_result（读取原始文本，执行对应动作，并从发送内容里剥离标签）
         → 发送给用户（干净文本，无标签）
```

### 与其他插件的兼容性

- **遗忘插件**（llm_amnesia）：对话历史按 `history.tag_mode` 处理（剥离 / 原样保留 / 摘要），遗忘/恢复操作均正常工作（写入历史的只是文本，不影响历史读写）
- **消息防抖插件**：戳一戳事件注入的文本会正常参与防抖合并
- **主动消息插件**（proactive_chat）：`[poke]` 标签可在主动消息中正常解析执行
- **TTS 插件**（tts_modify）：与 `<tts>` 标签完全兼容。TTS 插件（priority 13）先将 `<tts>...</tts>` 转换为 Record 语音组件，本插件在处理引用/撤回时会自动剥离残留的 `<tts>` 标签，避免标签文字泄漏给用户，同时语音组件在正确位置发送
- **分段回复**：`[recall]` 和 `[reply:ID]` 均支持分段场景，每段独立判断是否撤回

## 已知限制

- 流式输出模式下，`[poke]` 标签可能短暂出现在消息中（随后会被撤回并重发干净文本，需要 OneBot 实现支持 `get_friend_msg_history` API）
- 仅支持私聊场景，群聊中标签不会被处理

## 平台要求

- AstrBot 框架
- aiocqhttp（OneBot v11）适配器
- 支持 `friend_poke` API 的 OneBot 实现（如 NapCat、LLOneBot）
